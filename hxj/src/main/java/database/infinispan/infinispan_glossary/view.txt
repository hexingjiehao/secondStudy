1.infinispan的31个术语：全部是数据的一致问题。网络+数据库的组合
    1.两阶段提交。分布式中的同步相关
        ：是一种共识协议。用于原子提交或回滚分布式事务。
    2.原子性、一致性、隔离性、耐久性(ACID)
        ：是一组保证可靠处理数据库事务的属性。在数据库上下文中，对数据的单个逻辑操作称为事务。
    3.基本可用，软状态，最终一致(BASE)
        ：最终一致性是分布式系统基础，与ACID相反的极性，目的是提供高可用性和分区容忍度(AP)
    4.一致性、可用性和分区容限(CAP)定理
        :分布式计算定理，只能3选2。infinispan偏向CA，内部有一个最终一致模式（AP）
    5.一致性哈希
        ：将秘钥映射到服务器的一种算法。在稳定拓扑下，集群任意服务器都能最快的找到秘钥位置
    6.数据网格
        ：本地局域网上相互连接的一组服务器。对等方式工作。数据分区，总体存储容量，单个节点负载
    7.死锁
        ：指两个或两个以上相互竞争的动作都在等待另一个动作完成，因此两个动作都没有完成。
    8.分布式哈希表(DHT)
        ：分布式系统的一类。参与节点都能够快速定位key-value,维护键值映射关系。
    9.外部化
        ：是一个类，将对象marshall到字节数组和相反操作，类似于序列化
    10.热棒⚠️⚠️⚠️：️
        ：是一个协议。能够在发生拓扑更改的Infinispan服务器集群上处理故障转移。定期通知客户机集群的拓扑结构。
          它能使客户机在Infinispan服务器集群中对请求进行智能路由，服务器和客户端使用一致的散列算法。
    11.内存数据网格(IMDG)：
        :是一个特殊数据网格。每个服务器使用系统内存来进行数据的存储。效率提升。
    12.隔离级别
        ：定义一个操作在什么时候对其他操作可见。infinispan默认级别是可提交读
    13.JTA同步
        ：同步器是一个事务周期的监听器。一般用于刷新缓存。
    14.活锁
        ：和死锁类似，但是竞争的方式不一样。都想让其他线程先通过，结果整体没线程通过。
    15.Memcached
        ：内存缓存系统。infinispan使用Memcaches协议服务器，可以使memcached被自己替换。
    16.多版本并发控制(MVCC)
        ：数据库管理系统常用的方法。提供数据库并发访问，在代码中实现事务内存。
    17.近缓存
        ：在客户机中通过热棒协议与远程缓存通信时缓存数据的一种技术。这种技术有助于最小化检索数据的远程调用
    18.网络分区
        ：集群系统的多部分因为网络故障分开时，短期可自动修复，但可能造成数据不一致。解决办法：牺牲可用性/进入只读模式，更新处理网络分区
    19.非关系型的数据库
        ：高度优化的键值存储，一致性模型约束少。
    20.乐观锁定
        ：并发控制方法。多个事务互不影响完成的前提下，不锁定资源。提交时验证是否修改过，有则回滚。
    21.悲观锁定
        ：并发控制方法。单个线程对数据独占，用完后释放。
    22.提交读
        ：传统方式，只要没有写，就可以读取数据。infinispan，允许对数据副本进行写操作时可以读。
          使用READ_COMMITTED，事务中相同键的多次读取可以返回不同的结果
    23.关系数据库管理系统(RDBMS)
        ：基于关系模型的系统
    24.可重复读取
        ：传统方式，只要没有写，就可以读取数据，反之亦然。infinispan的定义是，每当修改一个条目时，都会保留先前的值。
          举例：
            cache.get("A") // returns 1
            cache.get("B") // returns 1

            Thread1: tx1.begin()
            Thread1: cache.put("A", 2)
            Thread1: cache.put("B", 2)
            Thread2:                                       tx2.begin()
            Thread2:                                       cache.get("A") // returns 1
            Thread1: tx1.commit()
            Thread2:                                       cache.get("B") // returns 2
            Thread2:                                       tx2.commit()
    25.表征态转移(ReST)
        ：是一个软件结构风格，基于http。组成由GET、PUT、POST、DELETE和HEAD
    26.分裂的大脑
        ：网络分区的通俗术语
    27.结构化查询语言(SQL)
        ：是编程语言，管理关系数据库数据
    28.滞后写
        ：是缓存存储更新模式。异步写入缓存存储
    29.倾斜写
        ：它是一个异常，出现在快照隔离中。infinispan可以检测这种异常。并且可以配置为在检测到写倾斜时回滚事务。
    30.连续写入
        ：是缓存存储更新模式，客户端更新缓存，底层缓存更新后，调用才会返回。
    31.XA资源
        ：是分布式事务的参与者。比如给定数据库和infinispan分布式事务，XA将infinispan和数据库定义为XA资源
